Termfolio v1.0.0 is the first public stable release of the dark, terminal-inspired Hugo theme for technical blogs, project portfolios, and repository-style personal sites.

## Highlights

- A filesystem-inspired interface with shell prompts, command-style labels, file-list navigation, and a dark-only terminal palette.
- Dedicated layouts for long-form posts and structured project pages, including stack, status, role, impact, repository, demo, licence, and navigation metadata.
- Category, tag, and series indexes and term pages, with descriptive term content and compatibility across the supported Hugo range.
- Optional search using either the built-in static index or locally served Pagefind assets.
- Table of contents, latest-content links, previous and next navigation, figures, asides, disclosure blocks, and external-link shortcodes.
- JSON-LD, Open Graph, Twitter Card, canonical URL, RSS, favicon, and web-app metadata.
- Privacy-conscious defaults with locally hosted JetBrains Mono, no analytics or tracking, and no third-party runtime assets. JavaScript is loaded only when optional search is enabled.

## Compatibility

- Requires Hugo `0.128.0` or newer.
- Standard Hugo is supported; Hugo Extended is not required.
- Continuous integration exercises standard Hugo `0.128.0` and `0.164.0` with warnings treated as errors.
- Hugo configuration may use YAML, TOML, or JSON.
- The interface is intentionally dark-only.

## Installation

### Git submodule

```bash
git submodule add https://github.com/joe-mccarthy/termfolio themes/termfolio
git -C themes/termfolio checkout v1.0.0
```

Record the resulting submodule reference in the parent site's repository so the installed theme version remains reproducible.

### Release archive

Download the source ZIP or tar.gz archive from this release, extract it to `themes/termfolio`, and configure Hugo to use the theme.

A minimal YAML configuration starts with:

```yaml
defaultContentLanguage: en-gb
theme: termfolio

markup:
  highlight:
    noClasses: false
```

Use the appropriate language identifier for your site. The README and the `docs/` directory in the source archive provide complete configuration, content-authoring, search, styling, development, contribution, and support guidance. [Read the full documentation](https://github.com/joe-mccarthy/termfolio#documentation).

## Upgrading from an unreleased checkout

There is no migration from an earlier public release. If the theme was installed from a branch or commit before `v1.0.0`:

- Pin the installation to the `v1.0.0` tag.
- Add `defaultContentLanguage`; current examples use it instead of relying on `languageCode` for the rendered page language.
- Compare site configuration with `example-config.yaml`, particularly syntax highlighting, navigation, taxonomies, search, and display parameters.
- Review site-owned layout overrides before updating because local overrides take precedence over theme templates.

## Search

Search is disabled by default. The built-in provider is suitable for smaller sites and creates a client-side index during the Hugo build. Pagefind is better suited to larger collections, but it must be installed separately and run against the generated `public/` directory after Hugo builds the site.

## Quality

The repository includes warning-free render checks, Markdown and link validation, ShellCheck, release-validator tests, and Lighthouse accessibility audits for the home, posts, projects, and search routes. The accessibility gate requires a score of at least `0.95` on each audited route.

## Known limitations

- Termfolio does not provide a light colour scheme.
- The built-in search index grows with published text; use Pagefind when that trade-off no longer suits the site.
- Distribution consists of GitHub's source ZIP and tar.gz archives. No binary or package-registry artifacts are produced.
